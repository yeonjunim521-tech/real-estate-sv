# 종합 부동산 데이터 플랫폼 고도화 설계

- 작성일: 2026-07-13
- 대상 저장소: `yeonjunim521-tech/real-estate-sv`
- 상태: 사용자 승인 완료 설계
- 구현 범위: 설계만 포함하며 이 문서 커밋은 운영 코드와 배포를 변경하지 않는다.

## 1. 목표

현재의 8종 월별 실거래 조회 대시보드를 공공데이터 기반 종합 부동산 분석 플랫폼으로 확장한다. 사용자는 부동산 유형과 관계없이 거래, 건축물, 토지이용, 시장 수급, 인구 및 입지 데이터를 같은 체계에서 조회하고 비교할 수 있어야 한다.

서비스의 핵심 차별점은 다음과 같다.

1. 실거래 흐름과 도시계획·용적률 정보를 한 화면에서 결합한다.
2. 일반 사용자는 쉽게 조회하고 투자자는 상세 근거까지 단계적으로 확인한다.
3. 모든 분석값에 기준일, 공식 출처, 신뢰도를 표시한다.
4. 공식 데이터가 없거나 매칭이 불확실한 값은 추정값으로 위장하지 않는다.
5. 프리미엄 기능은 베타로 검증한 후 결제를 도입한다.

## 2. 제품 원칙

- 공공기관·법령·지자체 공식자료를 우선한다.
- 취소 거래는 목록에는 표시하지만 가격·평당가·거래량 분석에서는 제외한다.
- 평균보다 이상치 영향을 덜 받는 중앙값을 대표 가격으로 사용한다.
- 부동산 유형별 면적 기준을 명확히 분리한다.
- 조례상 기본 상한과 해당 필지의 최종 적용 상한을 동일한 값처럼 표현하지 않는다.
- 부분적인 외부 API 장애가 전체 서비스 장애로 이어지지 않게 한다.
- 비로그인 사용을 허용하고 저장·동기화 시점에만 로그인을 유도한다.
- 관리자 버튼 숨김과 별개로 모든 관리자 요청을 서버에서 권한 검증한다.

## 3. 확정된 사용자 경험

### 3.1 기본 레이아웃

홈은 `분석 대시보드형`으로 구성한다.

1. 통합검색·지역 선택
2. 부동산 유형·기간·세부 필터
3. 핵심 지표
4. 가격·평당가·거래량 차트
5. 실거래 목록
6. 거래 선택 시 상세 규제 패널

데스크톱에서는 오른쪽 상세 패널을 사용하고 모바일에서는 하단 전체 화면 패널을 사용한다.

### 3.2 디자인 시스템

- 적응형 듀얼 테마
- 라이트모드 기본
- 다크모드 선택 가능
- 중립적인 네이비·블루 계열을 기본 색상으로 사용
- 취소·오류·주의·신뢰도 상태는 색상뿐 아니라 텍스트와 아이콘으로도 구분
- 데스크톱은 표·차트·패널, 모바일은 카드·탭·하단 상세창으로 재배치

### 3.3 전체 메뉴

| 메뉴 | 목적 |

| 통합 분석 | 검색, 핵심지표, 5년 차트, 실거래 목록 |

## 4. 지원 범위

### 4.1 부동산 유형

공식 데이터가 식별 가능한 범위에서 다음 유형을 지원한다.

- 아파트·주상복합
- 연립·다세대·빌라
- 단독·다가구
- 오피스텔
- 상가·점포·사무실
- 상업·업무용 빌딩
- 공장·창고·지식산업센터
- 숙박·의료·교육 등 비주거 건축물
- 토지·임야·농지
- 분양권·입주권
- 건축물대장 주용도 기반 기타 유형

국토교통부 실거래 API가 별도 거래 유형으로 제공하지 않는 세부 용도는 건축물대장 주용도와 거래 원천 유형을 결합해 분류한다. 이 경우 화면에 원천 거래 유형과 재분류 유형을 함께 기록한다.

### 4.2 분석 데이터

- 매매 및 공식 API가 제공하는 전월세
- 거래금액, 면적, 층, 계약일, 거래방식, 취소 여부
- 공급, 입주예정, 미분양, 분양
- 인구, 세대수, 전출입
- 공시지가, 공동주택가격
- 용도지역, 용도지구, 용도구역, 지구단위계획
- 건축물대장, 건폐율, 용적률, 연식, 층수, 주차
- 학군, 교통, 상권, 편의시설 등 공개 가능한 입지 데이터

매물, 중개사 네트워크, 독자 시세, 일부 경매·상권 및 자동설계 데이터는 공공데이터만으로 경쟁 서비스와 동일하게 제공할 수 없다. 향후 정식 제휴 또는 이용 허가가 확보된 경우 별도 모듈로 추가한다.

## 5. 주요 화면

### 5.1 통합 분석 대시보드

상단에 통합검색, 시·도/시·군·구/읍·면·동 선택, 기간, 유형 및 추가 필터를 둔다.

기간은 1개월, 3개월, 6개월, 1년, 3년, 5년을 제공한다.

핵심 지표는 다음과 같다.

- 유효 거래 수
- 취소 거래 수
- 중앙 거래가
- 평균 거래가
- 평균 또는 중앙 평당가
- 전년 대비 가격 변화
- 데이터 마지막 갱신 시각

차트는 월별 중앙 거래가, 평당가, 유효 거래량, 취소량, 면적 구간별 가격 분포를 제공한다.

### 5.2 실거래 목록

- 유효 거래와 취소 거래를 기본적으로 모두 표시한다.
- 취소 거래에는 취소 배지와 취소일을 표시한다.
- 요약에는 `전체 신고 / 유효 / 취소`를 분리해 표시한다.
- 모든 분석 계산은 취소 거래를 제외한다.
- 열 선택, 정렬, 페이지 크기, 모바일 카드 보기를 제공한다.

### 5.3 거래 상세 패널

- 거래금액, 계약일, 면적, 층, 건축연도
- 취소 상태와 취소일
- 직거래/중개거래 및 제공되는 중개사 소재지
- 동일 부동산의 최근 거래 이력
- 필지 위치와 용도지역 레이어가 포함된 미니 지도
- 용도지역, 용도지구, 용도구역
- 건축물대장상 현재 건폐율·용적률
- 지자체 조례상 기본 상한
- 지구단위계획·고도 제한 등 추가 규제
- 토지이음 등 공식 확인 링크

### 5.4 비교 분석

무료 사용자는 지역, 단지, 건물 또는 필지를 최대 3개까지 비교한다. 프리미엄 베타 사용자는 최대 10개까지 비교한다.

비교 지표는 가격 추세, 평당가, 거래량, 취소율, 공급, 인구, 용도지역, 현재 용적률 및 기본 상한이다. 서로 다른 부동산 유형 사이에서는 공통 정의가 성립하는 지표만 직접 비교한다.

### 5.5 빅데이터 지도

홈 대시보드와 별도의 메뉴로 제공한다.

- 실거래 가격·거래량
- 용도지역·용도지구·지구단위계획
- 공급·입주·미분양
- 인구·세대·전출입
- 공시지가·건축물 노후도
- 이후 공개되는 입지 데이터

지도는 탐색과 비교를 위한 수단이며 토지이용 규제의 법적 판단 화면으로 표현하지 않는다.

## 6. 데이터 원천과 결합

| 데이터 | 우선 원천 | 사용 목적 |
|---|---|---|
| 실거래·전월세 | 국토교통부 실거래가 Open API | 가격·면적·계약·취소·거래방식 |
| 법정동·필지 | 공식 행정구역·연속지적 데이터 | 전국 검색, PNU, 필지경계 |
| 용도지역·규제 | 국토교통부 토지이용계획정보·VWorld | 용도지역, 용도지구, 지도 레이어 |
| 건축물 | 국토교통부 건축HUB | 현재 건폐율·용적률·연식·용도 |
| 법령·조례 | 국가법령정보 공동활용 API | 국가 범위와 지자체 기본 상한 |
| 시장 수급 | 한국부동산원·국토교통부·지자체 공개자료 | 공급, 입주, 미분양, 분양 |
| 인구·이동 | 행정안전부·통계청 공개자료 | 인구, 세대, 전출입 |
| 입지 | 교육·교통·소상공인 관련 공식 공개자료 | 학군, 교통, 상권, 편의시설 |

### 6.1 식별체계

- 토지·단독·공장·일반 상업용은 PNU를 우선 식별자로 사용한다.
- 아파트·오피스텔·연립 등 집합건물은 주소, 건물명, 건축물대장 PK를 결합한다.
- 원천 API 지번이 마스킹되거나 복수 필지가 존재하면 자동 확정하지 않는다.
- 복수 후보는 사용자가 선택할 수 있게 하고 신뢰도를 낮춘다.

### 6.2 용적률 의미

화면에는 다음 값을 구분해 표시한다.

1. `현재 용적률`: 건축물대장에 기록된 값
2. `조례상 기본 상한`: 해당 지자체 도시계획조례의 용도지역별 기본값
3. `국가법령 최대범위`: 법률·시행령이 허용하는 큰 범위
4. `추가 규제`: 지구단위계획, 고도지구, 정비계획 등
5. `개발 여유 참고치`: 조례상 기본 상한에서 현재 용적률을 뺀 참고값

개발 여유 참고치는 실제 추가 건축 가능 면적이 아니다. 대지, 도로, 일조, 높이, 주차, 정비계획 등 추가 조건을 별도로 확인해야 한다.

### 6.3 신뢰도

- 높음: PNU, 건축물대장, 현행 조례가 정확히 일치
- 중간: 주소는 일치하지만 복수 필지 또는 복수 건축물 존재
- 낮음: 지번 마스킹, 원천 필드 누락, 조례 자동해석 불확실

모든 규제정보에는 기준일, 원천, 신뢰도, 공식 확인 링크를 표시한다.

## 7. 통계 기준

- 대표 가격은 중앙값을 기본으로 하고 평균을 보조로 제공한다.
- 취소 거래는 가격, 평당가, 거래량, 시그널 계산에서 제외한다.
- 아파트·오피스텔·연립은 전용면적을 사용한다.
- 토지는 거래면적을 사용한다.
- 단독·다가구는 연면적과 대지면적 기준을 각각 제공한다.
- 상업·공장·창고는 건물면적과 토지면적 기준을 분리한다.
- 서로 다른 면적 기준의 평당가를 하나의 순위로 합치지 않는다.
- 통계에 사용한 거래 건수와 제외 건수를 함께 표시한다.

## 8. 기술 아키텍처

### 8.1 선택안

Cloudflare 중심 하이브리드 구조를 사용한다.

- 프런트엔드: React + TypeScript + Vite
- 정적 파일 및 API: Cloudflare Worker
- 공공데이터·통계 저장: Cloudflare D1
- 인증: Supabase Auth의 Google·카카오 OAuth
- 예약 작업: Cloudflare Cron Trigger
- API 비밀값: Cloudflare Secret

현재의 Cloudflare Worker Static Assets 배포 방식을 유지한다.

### 8.2 서버 모듈

- `transactions`: 실거래·전월세 수집, 페이지네이션, 정규화
- `properties`: 단지·건물·필지 식별과 검색
- `regulations`: 토지이용, 건축물대장, 조례 결합
- `market`: 공급, 입주, 미분양, 분양, 인구, 이동
- `analytics`: 중앙값, 평당가, 거래량, 시그널
- `accounts`: 관심목록, 저장 분석, 계정 동기화
- `premium`: 베타 기능과 기능별 접근 통제
- `admin`: 관리자 통계, 권한, 기능 스위치
- `jobs`: 예약 갱신, 데이터 품질 검사

### 8.3 주요 데이터 영역

- transactions
- properties
- parcels
- building_profiles
- zoning_profiles
- ordinance_rules 및 적용일
- monthly_market_stats
- supply_and_demand_stats
- saved_analyses 및 watchlists
- user_roles 및 linked_identities
- beta_access 및 feature_flags
- aggregated_product_events
- data_quality_issues

구현 계획에서 실제 테이블 이름과 인덱스, 보존기간을 확정한다.

## 9. 5년 조회와 갱신

- 원천 API의 `totalCount`와 페이지를 사용해 100건 제한 없이 수집한다.
- 최초 조회 시 최근 월부터 표시하고 과거 5년은 단계적으로 적재한다.
- 최근 3개월은 매일 갱신한다.
- 4~12개월은 주간 갱신한다.
- 1년 이전은 월간 갱신한다.
- 인기 지역과 최근 조회 데이터만 예약 갱신해 외부 API 호출량을 제어한다.

## 12. 관리자 페이지

| 영역 | 지표 및 기능 |
|---|---|
| 종합현황 | 방문, 검색, 상세, 비교, 저장, 베타 신청 |

| 성능·비용 | 응답시간, D1 사용, 외부 API 호출, 캐시 상태 |

## 13. 광고

초기에는 사용자 분석 흐름을 방해하지 않는 광고만 사용한다.

## 14. 오류 처리

| 상황 | 동작 |
|---|---|
| 일부 거래 API 실패 | 성공 유형을 먼저 표시하고 실패 유형만 재시도 |
| 5년 데이터 수집 중 | 최근 결과와 진행 상태를 먼저 표시 |
| 토지이용 매칭 실패 | 거래는 표시하고 규제정보를 확인 필요로 표시 |
| 건축물대장 누락 | 현재 용적률을 추정하지 않고 자료 없음 표시 |
| 조례 해석 불확실 | 국가 범위·원문 링크·낮은 신뢰도만 표시 |
| 로그인 장애 | 비로그인 모드와 기기 저장으로 계속 사용 |
| 오래된 데이터 | 마지막 갱신 시각과 갱신 중 상태 표시 |
| 관리자 권한 없음 | UI 숨김 및 API 403 응답 |

오류 메시지는 데이터 없음, 외부 API 장애, 입력 오류, 권한 오류를 구분한다.
