# 부동산 종합 분석 플랫폼 1단계 구현 계획

> 기준일: 2026-07-15  
> 범위: 기존 계산기 흐름과 운영 D1 스키마를 보존하면서 분석 기반을 순차 확장한다.

## 목표

전국 단위 부동산 식별·거래·공법 데이터를 쌓고, 5년 시계열과 3개 대상 비교가 가능한 1단계 기반을 완성한다.

## 공통 성공 기준

- 기존 단일 월 실거래 조회가 동일하게 동작한다.
- 수집 데이터의 출처, 상태, 최종 갱신 시각을 확인할 수 있다.
- 수집 실패가 기존 정상 스냅샷을 손상하지 않는다.
- 타입 검사, 전체 테스트, 프런트 문법 검사, 배포용 빌드가 통과한다.
- 운영 D1 변경과 배포는 별도 승인 후 실행한다.

## Task 1. 수집 실행 이력과 갱신 상태 연결 — 이번 구현 묶음

**파일**

- 수정: `src/transaction-store.ts`
- 생성: `src/transaction-ingestion.ts`
- 수정: `src/worker.ts`
- 수정: `test/transaction-store.test.ts`
- 수정: `test/worker-storage.test.ts`

**구현**

1. 완료된 수집의 시작·완료 시각, 페이지 수, 거래 수를 `ingestion_runs`에 기록한다.
2. 페이지 저장과 실행 이력 기록을 하나의 백그라운드 작업으로 묶는다.
3. 실시간 응답에 `X-Data-Source`, `X-Data-Status`, `X-Data-Fetched-At` 헤더를 추가한다.
4. D1 fallback 응답의 기존 출처·갱신 헤더와 상태 헤더를 일치시킨다.

**검증**

- 저장 SQL과 바인딩 값을 단위 테스트로 확인한다.
- 응답을 지연하지 않고 `waitUntil`에 저장 작업이 등록되는지 확인한다.
- 응답 헤더와 페이지·실행 이력 동시 저장을 회귀 테스트로 확인한다.

**위험**

- D1 쓰기 실패는 실거래 응답을 실패시키지 않고 로그로만 남긴다.
- 이번 묶음은 성공한 수집 완료 이력부터 기록한다. 실패·부분 수집 분류는 관리자 품질 단계에서 확장한다.
